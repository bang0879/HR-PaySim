# Loyalty Tax Relationship Review Prototype

Standalone static prototype for testing the `loyalty_tax` Relationship Review grammar.

Open `index.html` directly in a browser. No dev server is required.

## Status

This is an isolated grammar reference prototype.
It is not production UI.
It should not modify parser, detector, routing, session, or production app flow.

## Core Frame

`loyalty_tax` is treated as a cohort/pattern finding, not a single one-to-one relationship.

The screen may show one representative comparison so the founder can understand the pattern quickly, but the finding is about where long-tenured employees sit compared with recent hires.

## Three-Part Grammar

1. 왜 이 패턴을 먼저 봅니까?
2. 이 차이는 어떻게 생겼다고 보십니까?
3. 다음 기준으로 어떻게 남깁니까?

## Primary Options

1. 최근 채용 시장 영향입니다
2. 초기 구성원 인상 반영이 늦었습니다
3. 역할이나 성과 차이로 설명됩니다
4. 데이터를 더 확인해야 합니다

## Memo Grammar

The memo preview stays in three sections:

1. 현재 패턴
2. 대표가 선택한 이유
3. 다음 기준

No secondary chips are included in this first prototype.

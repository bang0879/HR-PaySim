# Shadow Band Relationship Review Prototype

Standalone static prototype for testing the `shadow_band` Relationship Review grammar.

Open `index.html` directly in a browser. No dev server is required.

## Status

This is an isolated grammar reference prototype.
It is not production UI.
It should not modify parser, detector, routing, session, or production app flow.

## Core Aha

제도가 없다고 구조가 없는 것은 아니다.

The prototype shows that a founder may have no formal compensation band, while the roster still reveals repeated compensation zones that need language.

## Three-Part Grammar

1. 왜 이 분포를 먼저 봅니까?
2. 이 구간은 어떻게 생겼다고 보십니까?
3. 다음 기준으로 어떻게 남깁니까?

## Primary Options

1. 의도한 구간입니다
2. 자연스럽게 생긴 구간입니다
3. 처음 보는 구조입니다
4. 데이터가 더 필요합니다

## Memo Grammar

The memo preview stays in three sections:

1. 현재 분포
2. 대표가 선택한 설명
3. 다음 기준

The memo translates the selected distribution explanation into a candidate compensation standard. It does not capture raw free text, names, quotes, or sensitive employee-level details.

## Difference From Pay Inversion

- Pay inversion reviews a relationship between two compensation rows.
- Shadow band reviews a distribution pattern across a role group.
- The founder is not explaining why one person earns more; the founder is deciding whether an unnamed compensation zone should become explicit language.
